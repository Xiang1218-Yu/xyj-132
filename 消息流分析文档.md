## 三、核心业务消息流详细流程

### 3.1 页面预览请求流程（最长路径）

```mermaid
sequenceDiagram
    participant U as 用户
    participant CS as Content Script<br/>(网页内)
    participant MSG as Chrome Runtime
    participant BW as Background SW<br/>(后台)
    participant CACHE as 内存缓存<br/>(LRU Map)
    participant NET as 目标网站<br/>(HTTP)
    participant STORE as chrome.storage

    U->>CS: 悬停/点击链接
    CS->>CS: 1. isValidUrl() 校验URL
    CS->>CS: 2. matchPreviewRule() 匹配规则
    CS->>CS: 3. getLinkType() 判断类型
    CS->>CS: 4. evaluateUrlSecurity() 安全评估<br/>(纯本地计算)
    
    alt 类型为 webpage/video-site/audio-site
        CS->>MSG: sendMessage({action:'fetchPageInfo', url})
        MSG->>BW: onMessage 路由分发
        
        BW->>CACHE: getFromCache(url) 查缓存
        alt 缓存命中
            CACHE-->>BW: 返回缓存数据
            BW-->>MSG: sendResponse({success:true, data})
        else 缓存未命中
            BW->>NET: fetch(url) + AbortController<br/>(8s超时)
            Note over NET: ⚠️ 阻塞点 1: 网络IO
            alt 请求成功
                NET-->>BW: 返回 HTML
                BW->>BW: extractMetaTags() 解析元数据
                BW->>CACHE: setToCache(url, data)<br/>(LRU淘汰)
                BW-->>MSG: sendResponse({success:true, data})
            else 请求失败/超时
                NET-->>BW: Error / AbortError
                BW-->>MSG: sendResponse({success:false, error})
            end
        end
        
        MSG-->>CS: 回调 response
        alt response.success
            CS->>CS: renderRichPreview() 渲染预览
        else 失败
            CS->>CS: handleLoadError() 错误处理
            CS->>CS: 重试(最多maxRetries次)
            CS->>CS: → renderFallbackPreview() 降级
        end
    else 类型为 image/video/audio
        CS->>NET: 直接通过<img>/<video>/<audio>加载
        Note over NET: ⚠️ 阻塞点 2: 媒体加载
        CS->>CS: 超时 + 重试 + 降级UI
    end
    
    CS->>MSG: sendMessage({action:'addPreviewHistory', item})<br/>(异步, 不等待响应)
    MSG->>BW: addPreviewHistory()
    BW->>STORE: chrome.storage.local.set
```

### 5.3 降级策略 (Fallback)

当所有重试均失败后，系统会渲染 Fallback 预览，确保用户始终能看到有意义的内容：

```
┌─────────────────────────────────────────────────┐
│  🔗 原始降级策略 (renderFallbackPreview)         │
├─────────────────────────────────────────────────┤
│  ✓ 仍然显示 URL / hostname / favicon            │
│  ✓ 仍然显示安全评估徽章                          │
│  ✓ 提供"在新标签页打开"按钮                      │
│  ✗ 不显示标题/描述/OG图片（无法获取）            │
│  ✗ 不提供嵌入预览                                │
└─────────────────────────────────────────────────┘
```

**触发条件**：
1. `retryCount >= LOAD_CONFIG.maxRetries`
2. 网络请求返回无法恢复的错误
3. 用户等待超过超时阈值

## 八、潜在风险与改进建议

| 风险等级 | 问题描述 | 影响 | 建议 |
|---------|---------|------|------|
| 🔴 **高** | `Popup → Tab sendMessage` 无超时无错误回调 | 用户点击历史项无反应，无任何提示 | 建议加上 `runtime.lastError` 检测和超时机制，失败后提示用户 |
| 🟠 **中** | Background 的 `fetchPageInfo` 使用 8s 硬超时，无退避重试 | 网络波动时预览成功率低 | 建议在 fetch 层增加 1 次自动重试，配合指数退避 |
| 🟠 **中** | `chrome.storage` 操作无错误回调检查 | 磁盘满/配额超限会静默失败 | 建议统一封装 storage 方法，检查 `chrome.runtime.lastError` |
| 🟡 **低** | `completed` 状态锁防重复回调，但无消息去重 | 极端情况下重复请求浪费带宽 | 可增加 requestId 去重 |
| 🟡 **低** | 内存缓存无持久化，SW 重启即清空 | 冷启动性能差 | 可考虑定期写入 IndexedDB |
